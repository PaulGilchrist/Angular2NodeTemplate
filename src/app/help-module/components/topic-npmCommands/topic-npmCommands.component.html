NPM and NG Commands

Angular CLI has several useful commands for creating and building applications. These commands can be run from the console by preceding them with the name “ng” (example: ng new [myNewApp])

CommandDescription
newCreates a new project from a template that follow Angular best practices and styles.
serveBuilds the application outputing it to memory only, then launching and watching the application for code changes, automatically re-compiling and refreshing the browser as needed.
buildBuilds the application following the options listed in angular.json or passed on the command line, based on the environment configuration passed in. A default new project has a development environment (default), and a production environment. Other envirnments like QA and staging, can be added to angular.json as needed.
testBuilds the test configuration as defined in angular.json, and discovers and launches all 'spec' files reporting on the results of each test.
lintAnalyses the source code and compared against rules defined in tslint.json reporting on any found issues.
e2eRuns all the end to end tests located in the e2e folder reporting on any found issues.

Node Package Manager (NPM) can also execute scripts defined in the package.json file but it is best to use the Angular commands whenever possible. These commands can be run from the console by preceding them with the name “npm” (example: npm install)

CommandDescription
installReads the “package.json” file, identifies all node packages required by the project, then pulls them down from the Internet and places them (including all dependencies) into the “node_modules” folder. Packages are not added to version control, so this command should be run whenever the packages within “package.json” changes. This command is idempotent so it is safe to run at any time and will not re-download packages that already exist and have not changed.
